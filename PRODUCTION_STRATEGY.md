# YouTube Production Strategy: Prompt Saver Build Series

## 1. Lead Variation Selection

### Recommendation: **Variation 3 (Mistake-Prevention) as Lead Video**

**Strategic Reasoning:**
- Your audience (builders, business owners, automation enthusiasts) responds to **pragmatic, actionable content** over storytelling
- "Don't waste time" messaging creates urgency and psychological investment
- Mistake-prevention positioning establishes you as someone who's "been through it"
- Highest estimated CTR (7-9%) and viral potential
- Creates natural bridge to Variation 1 (full story) as follow-up

**Why NOT lead with Variation 1:**
- Story-first requires more emotional trust with new audience
- Better as video 2 when viewers are already invested
- Works better if posted after initial traction

**Why NOT lead with Variation 2:**
- Too technical for cold audience
- Perfect as deep-dive after they're familiar with the basics
- Better as a "for advanced builders" video later

### Video Release Sequence

**Week 1 - Video 1 (Variation 3): "7 Mistakes That Stop People Building Chrome Extensions"**
- Lead with mistake-prevention angle
- Fast-paced, practical, high value density
- Goal: Hook audience, establish authority, get subscribers

**Week 2 - Video 2 (Variation 1): "How I Built Prompt Saver in 8 Hours (The Real Journey)"**
- Tell the full story for viewers who watched Video 1
- Deeper dive into the actual process
- Create playlist: "Building Tools with Claude Code"
- Goal: Deepen relationship, show authentic process

**Week 3 - Video 3 (Variation 2): "Chrome Extension Architecture: The Systems Thinking Behind Prompt Saver"**
- Technical deep-dive for builders who want to understand the scaffolding
- Most advanced audience at this point
- Goal: Establish technical authority, attract serious builders

## 3. Screen Recording Technical Setup

### Software Required
- **Screen Recorder:** ScreenFlow (Mac), OBS (cross-platform), or Camtasia
- **Audio:** Use your mic, record separately and sync
- **Editing:** Adobe Premiere Pro, DaVinci Resolve (free), or Final Cut Pro
- **Graphics:** Canva Pro, Adobe Illustrator, or design elements

### Mac Screen Recording Setup (Recommended)
1. Close all unnecessary applications
2. Set display to 1920x1080 resolution
3. Zoom to 125% in text editor (readable on video)
4. Set Terminal/editor theme to dark mode
5. Open ScreenFlow
6. Select "Record from screen" â†’ "Don't show cursor"
7. Start recording
8. Speak clearly, pause between sections
9. Stop recording after all content

### Recording Checklist
- [ ] Room is quiet (close windows, door)
- [ ] Microphone is charged and positioned
- [ ] Screen is clean (no sensitive files visible)
- [ ] Resolution set to 1920x1080
- [ ] Font size is readable (14+ point)
- [ ] Browser/editor tabs are closed except working file
- [ ] Phone is on silent
- [ ] Recording software tested (do 30-second test first)
- [ ] Audio levels are set correctly (-6dB is ideal)
- [ ] You've read through the script once

### Recording Tips
1. **Do multiple takes:** Record the hook 3-5 times, pick the best
2. **Pause naturally:** Don't talk continuouslyâ€”add pauses for editing
3. **Speak clearly:** Imagine talking to one person across a table
4. **Point during explanations:** Use cursor highlight to draw attention
5. **Vary pace:** Slow for complex parts, faster for obvious stuff
6. **Spell names:** When mentioning Claude Code features, spell them visually
